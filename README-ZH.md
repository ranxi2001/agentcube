## 概览

AgentCube 是 Volcano 社区中的一个拟议子项目，目标是扩展 Volcano 的能力，使其能够原生支持和管理 AI Agent 工作负载。AI Agent 正在生成式 AI 和大语言模型（LLM）应用中快速出现，它们对运行时基础设施提出了不同于传统任务的新要求。

现有的工作负载管理模式，以及当前常见的批处理和推理系统，难以充分满足这类持续交互、需要状态保留、间歇活跃且会话较长的工作负载需求。

AgentCube 旨在为 AI Agent 提供专门的控制面和数据面组件，重点关注：

1. **极低延迟调度**：面向快速启动和交互式响应进行优化。
2. **有状态生命周期管理**：通过智能休眠和恢复机制提升资源效率。
3. **高密度资源利用**：在保证性能隔离的前提下进行更高效的装箱调度。
4. **命令式 API**：为 Agent 执行提供同步、命令式的 API 使用体验。

## Volcano 背景

Volcano 是基于 Kubernetes 的云原生批量计算引擎，基于华为云在 AI、大数据领域的深厚业务积累，补齐了 Kubernetes 在面向 AI、大数据、高性能计算等批量计算任务调度、编排等场景下的短板。

Volcano 向下支持鲲鹏、昇腾、X86 等多元算力，向上使能 TensorFlow、Spark、华为 MindSpore 等主流行业计算框架，让数据科学家和算法工程师能够充分享受到云原生技术带来的高效计算与良好体验。

## 为什么需要 AgentCube

Volcano 是云原生生态中的高性能批调度系统，适合管理复杂、计算密集型工作负载。AI Agent 应用代表了下一代 AI 工作负载，具有一些独特特征：

* **间歇活跃**：空闲时需要快速释放资源，重新交互时又要快速恢复。
* **高度延迟敏感**：为了获得良好的用户体验，需要尽可能低的响应延迟。
* **状态持久性**：需要在长时间、多轮交互的会话中保留上下文和状态。

引入 AgentCube 可以让 Volcano 完善对 AI 全生命周期的支持，使用户能够在 Kubernetes 上高效编排和管理 AI Agent 工作负载。对于这类“突发式”的 Agent 应用，AgentCube 有助于提升管理效率，并优化集群资源利用率。
